<Warning>
**Security-Critical Feature**: HandoffToolPolicy prevents privilege escalation in multi-agent deployments. Always test your handoff security boundaries.
</Warning>

## Overview

HandoffToolPolicy enforces tool security boundaries when one agent hands off tasks to another agent. This prevents privilege escalation where a limited agent could gain access to powerful tools through handoff.

## Default Security Mode

<Note>
**Secure by default**: The default `intersect` mode ensures target agents only receive tools that both source and target agents share.
</Note>

```python
from praisonaiagents import Agent, handoff

# Gatekeeper with limited tools
gatekeeper = Agent(name="Gatekeeper", tools=[web_search])

# Automation agent with powerful tools
automation = Agent(name="Automation", tools=[web_search, execute_code, file_system])

# Secure handoff (default)
secure_handoff = handoff(automation)

# Result: automation only gets [web_search] - the intersection!
```

## Legacy Passthrough Mode

For backward compatibility, you can opt into the legacy behavior:

```python
# Legacy mode - target keeps full toolset
legacy_handoff = handoff(
    automation,
    tool_policy_mode="passthrough",
    blocked_tools=["execute_code"]  # Can still block specific tools
)
```

## Tool Policy Configuration

<Tabs>
  <Tab title="Simple API">
    ```python
    # Via handoff() function parameters
    handoff(
        agent=target_agent,
        tool_policy_mode="intersect",  # or "passthrough"
        blocked_tools=["dangerous_tool"]
    )
    ```
  </Tab>
  
  <Tab title="Advanced Config">
    ```python
    from praisonaiagents import HandoffConfig, HandoffToolPolicy
    
    config = HandoffConfig(
        tool_policy=HandoffToolPolicy(
            mode="intersect",
            blocked_tools=["execute_code", "shell_access"]
        )
    )
    
    handoff_instance = Handoff(agent=target, config=config)
    ```
  </Tab>
</Tabs>

## Migration Guide

<Info>
No migration required! Your existing handoff code is now secure by default.
</Info>

If you need the old behavior for some reason:

```python
# Restore legacy behavior (not recommended)
handoff(agent, tool_policy_mode="passthrough")
```

## Implementation Details

The security boundary is enforced at multiple layers:

1. **Tool Intersection**: `_compute_effective_tools()` computes safe tool sets
2. **Chat Boundary**: `agent.chat(tools=effective_tools)` enforces restrictions  
3. **Format Layer**: Distinguishes `tools=[]` (deny) vs `tools=None` (inherit)

This provides defense-in-depth against privilege escalation attacks.
